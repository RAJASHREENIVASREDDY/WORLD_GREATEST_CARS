eXtensible Markup Language — storing and fetching car data
What is XML?

XML (eXtensible Markup Language) is used to store and transport data. Unlike HTML, XML tags are custom — you define them yourself. In this project, all car data is stored in cars.xml.

cars.xml — project data file
<?xml version="1.0" encoding="UTF-8"?>
<CarCollection>

  <Country id="germany">
    <n>Germany</n>
    <Flag>🇩🇪</Flag>
    <IconicCar>
      <Model>Porsche 911 GT3</Model>
      <Brand>Porsche AG</Brand>
      <TopSpeed>318 km/h</TopSpeed>
      <Horsepower>510 hp</Horsepower>
      <Fact>In production for over 60 years</Fact>
    </IconicCar>
  </Country>

  <Country id="italy">

  </Country>

  <!-- ... more countries ... -->

</CarCollection>

How It Works — JavaScript reading XML
xml.html — JavaScript
function fetchCarXML() {
  // Step 1: Create XMLHttpRequest object
  var xmlhttp = new XMLHttpRequest();

  // Step 2: Open and send request to cars.xml
  xmlhttp.open("GET", "cars.xml", false);
  xmlhttp.send();

  // Step 3: Parse the XML response
  var xmlDoc = xmlhttp.responseXML;
  var countries = xmlDoc.getElementsByTagName("Country");
  var key = document.getElementById("xml-key").value;

  // Step 4: Loop through and find matching country
  for (var i = 0; i < countries.length; i++) {
    if (countries[i].getAttribute("id") === key) {
      // Step 5: Display the car data
      document.getElementById("xr-car").textContent =
        countries[i].getElementsByTagName("Model")[0].childNodes[0].nodeValue;
    }
  }
}
XML Rules to Remember
RuleExample
Every tag must be closed<Model>Porsche</Model>
Tags are case-sensitive<Model> ≠ <model>
Must have one root element<CarCollection> ... </CarCollection>
Attribute values need quotesid="germany"
First line is declaration<?xml version="1.0"?>
